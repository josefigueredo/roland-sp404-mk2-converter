For the **Roland SP-404MKII**, the SD card is **not just a backup device**, but it is also **not a live streaming hard drive** like on an MPC.

It has a hybrid role.

---

# 1. What the SD Card Actually Does

The SD card is used for:

## ✅ 1. Import Source

You place WAV files on the SD → then IMPORT them into a project.

After import:

* Samples are copied into the SP’s internal project memory.
* They do **not stream directly from SD during playback**.

---

## ✅ 2. Project Storage

Projects themselves are saved on the SD card.

A project contains:

* Pad assignments
* Sample references
* Patterns
* FX settings
* Resamples

So the SD is where projects live.

---

## ✅ 3. Backup / Export

You can:

* Backup entire projects
* Export samples
* Move projects between machines
* Archive work

---

# 2. What the SD Card Does NOT Do

It does NOT:

* Stream long audio files directly (like MPC or DAW)
* Act as infinite sample RAM
* Replace internal performance memory
* Load instantly like a computer disk browser

When you import a sample:

> It becomes part of the active project.

---

# 3. Internal Memory vs SD (Conceptually)

Think of it like this:

| Component    | Role                        |
| ------------ | --------------------------- |
| Internal RAM | Active performance memory   |
| SD Card      | Project disk + sample vault |

The SP loads what it needs from SD into the active project.

---

# 4. Practical Example

You copy 500 SFM drum hits to:

```
ROLAND/IMPORT/
```

Inside the SP:

* You import 16 kicks into Project 1.
* Only those 16 become active in that project.
* The other 484 stay unused on SD.

---

# 5. Why This Matters For You

Since you:

* Use Maschine as your “main brain”
* Want performance layering live

Your optimal flow is:

SD = curated vault
Project = performance kit
Maschine = deep library

---

# 6. Capacity Reality

The SP-404MKII supports large SD cards (up to 1TB), but:

* You don’t want thousands of files inside IMPORT.
* Browsing becomes slow and annoying.
* Performance flow suffers.

Curation matters more than capacity.

---

# 7. Mental Model

It behaves closer to:

* A hardware groovebox with project-based memory
  Not like:
* A DAW with direct disk streaming.

---

# TL;DR

The SD card on the SP-404MKII:

* Stores projects
* Stores sample files for importing
* Handles backup/export

But playback happens from project memory — not as live disk streaming.
